# Location proofs

A location proof adds **evidence** to a location record. Where a [signed location record](/concepts/location-records) proves that someone *claimed* a location, a location proof provides independently evaluable evidence that the claim corresponds to physical reality.

This is Astral's core innovation: composable, multi-factor location proofs that bundle evidence from independent [proof-of-location systems](/concepts/pol-systems) into a single verifiable artifact.

## Why location proofs matter

Most location data today is self-reported. A device says "I'm at these coordinates" and the receiving system takes it on faith. This works until the stakes are high enough to justify forgery — and the cost of forging GPS coordinates is essentially zero.

Location proofs change the economics. By requiring evidence from independent proof-of-location systems, they raise the cost of forgery. The goal is not absolute certainty (which is rarely achievable for physical location), but making the cost of a convincing forgery exceed the value of the fraud.

## The certainty spectrum

Not all location proofs are equally strong. The level of assurance depends on the number, diversity, and quality of the evidence sources:

| Level | Description | Example |
|-------|------------|---------|
| **L0** | No evidence — bare self-reported claim | Raw GPS coordinates from an app |
| **L1** | One externally sourced piece of evidence | GeoIP lookup confirming approximate region |
| **L2** | One substantive strategy or 2-3 lighter-weight sources | ProofMode device attestation with sensor readings |
| **L3** | Multiple independent sources, at least one substantive | ProofMode + WitnessChain latency triangulation |
| **L4** | Diverse high-integrity signals with strong cryptography | Multiple hardware-backed sources + network verification + cross-correlation |
| **L5** | Nation-state resistance | Advanced cryptographic proofs + multiple cross-domain endorsements (research frontier) |

Most real-world applications operate between L1 and L3. Higher levels require proof-of-location systems that don't yet exist at scale — this is an active area of research.

## Key principles

**No one-size-fits-all.** Different applications need different levels of assurance. A social check-in app has different requirements than a land title registry. Location proofs are designed to be composable so applications can choose the right level for their use case.

**Probabilistic, not deterministic.** Location proofs produce credibility scores, not binary verdicts. The evidence either supports the claim strongly, weakly, or not at all — and applications decide what threshold they need.

**Compose for the use case.** A single location stamp from one proof-of-location system is a valid location proof. Multiple location stamps from independent systems are stronger. The architecture supports both without requiring a minimum.

**Forgery cost principle.** The cost of forging a location proof should exceed the economic value of the transaction it underpins. A $10 check-in reward needs less forgery resistance than a $10M land title transfer.
